# Mise à jour de la documentation

Mettre à jour la documentation afin de s'assurer que :

- Les contenus reflètent bien d'éventuels changements dans le code ou d'autres sources
- Les informations contenues dans la documentation reflètent bien la réalité du projet
- Les contenus ne comportent aucun lien brisé
- Les contenus ne comportent aucune information obsolète ou dépréciée
- Les sources markdown sont correctement formattés

## Informations concernant la documentation

- Les contenus de la documentation se trouvent sous /doc
- La documentation est propulsée par mkdocs, sauf le répertoire /doc/presentation qui est propulsé par Marp.
